Git简介

Git是一个分布式版本控制软件，最初由Linus Benedict Torvalds创作，以C语言开发，于2005年以GPL发布。
最初目的是为更好地管理Linux内核开发而设计。

集中式VS分布式

  • 集中式
    通过中央服务器进版本的存取，需要全程联网
  • 分布式
    每台电脑都是各自服务器

git以仓库作为管理单位

缓存区


git add把文件修改添加到缓存区
git commit把缓存区内容提交到当前分支

撤销修改

对于还未被放到缓存区文件的修改：撤销至未修改的状态
对于已放到缓存区文件的修改：撤销至刚放到缓存区的状态

删除

先用rm删除文件，然后用git rm删除仓库中的文件，最后git commit提交

如果是用rm误删了本地的文件，可以用git checkout从仓库中恢复
如果是用git rm误删了仓库文件，则用git reset恢复版本

git clone只能看到master分支，若想在其他分支上开发，则需先创建对应的本地分支

分支管理

分支可在不影响主线的情况下继续工作

  1. 在只有一个分支时，master指向最的提交，HEAD指向master。没提交一次master都会向前移动一步
  2. 创建新分支dev时，dev指向与master相同的提交，HEAD指向dev表示当前分支在dev
  3. 在dev分支下提交修改后dev向前移动而master不变
  4. 在dev分支上的工作完成后，通过把master指向dev来完成合并
  5. 通过删除dev指向来删除dev分支

分支冲突

当只有分支有修改，master没有修改时，git可执行fast merge,不会产生冲突
当都发生了修改时，则可能发生冲突，此时需先解决冲突再进行合并

  • 查看合并情况

分支管理

使用fast merge时，在删除掉分支后会丢失分支信息
若警用fast merge，则git会在合并时生成一个新的commit，从而可以在分支历史上看到分支信息

然后在git log中可看到分支历史

在实际开发中

  • master分支应稳定，仅用于发布新版本，不在上面直接开发
  • dev分支用于开发

多人协作

  1. clone仓库
  2. 建立分支的本地分支
  3. push本地分支修改到远程仓库

*如果git pull时发生no tracking information，则表明本地分支和远程分支没有建立链接
